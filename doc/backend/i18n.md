# Translating the Tracim Backend

Translation is done using the PO files generated by Babel.  
JSON files are generated from these files at [locale subdir](../tracim_backend/locale).

## Add a new language

If you want to add support for a new language, extracts the strings to translate and initialise the
new language. For instance, for the German language:

```bash
  python3 setup.py extract_messages
  python3 setup.py init_catalog -l "de"
```

`de` is iso639 for German langage. See
[the list of ISO_639-1_codes](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes) to determine
the language code to use for your language.

## Update Translations

To update the PO translation files, run:

```bash
  python3 setup.py extract_messages
  python3 setup.py update_catalog
```

## Translate

Translate by editing PO files (e.g. `tracim_backend/locale/_de_/LC_MESSAGES/tracim_backend.po`).

## Generate the JSON files

Install `i18n-conv`:

```bash
  sudo npm install "i18next-conv@<8" -g
```

Then, in the backend folder, run:

```bash
  ./update_i18n_json_file.sh
```

## Best Practices

- Do not edit the generated JSON file. Your work would be overwritten and the JSON files are not
tracked by git.
- English translation = translation key.

To avoid multiple versions of the English translation, default translation key are the English one.

__No translation should be done in English po files.__
